# 开源社区治理与运营建议（简版）

## Issue Triage
- 建议建立 triage 标签体系：bug/feature/question/priority 等
- 设定首响SLA：例如 24h 内首次响应
- 引入轮值机制：每周指定 1-2 名维护者负责 triage

## 新贡献者引导
- 维护 good first issue / help wanted
- 提供贡献指南、开发环境一键启动脚本
- 建立 mentor 机制：新人PR由导师跟进一次完整流程

## Review 负载治理
- 监控 PR 积压与平均review时延
- 扩大 reviewer 池：邀请活跃贡献者成为 reviewer
- 对关键模块设置 CODEOWNERS
